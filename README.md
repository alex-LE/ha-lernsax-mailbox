# LernSax Mailbox for Home Assistant

[![HACS Validation](https://github.com/alex-LE/ha-lernsax-mailbox/actions/workflows/hacs.yml/badge.svg)](https://github.com/alex-LE/ha-lernsax-mailbox/actions/workflows/hacs.yml)
[![Hassfest](https://github.com/alex-LE/ha-lernsax-mailbox/actions/workflows/hassfest.yml/badge.svg)](https://github.com/alex-LE/ha-lernsax-mailbox/actions/workflows/hassfest.yml)

Custom Home Assistant integration for LernSax mailbox monitoring.

This integration polls the LernSax mailbox of one or more accounts and exposes sensors for unread messages, so you can build your own Home Assistant automations, dashboards, and notifications.

## Features

- UI setup via Config Flow
- One config entry per LernSax account
- Multiple accounts supported
- Sensor for unread message count
- Binary sensor indicating whether unread mail exists
- Configurable polling interval
- Designed for automations instead of built-in notifications

## Current scope

This integration currently focuses on mailbox state sensors.

The official LernSax / WebWeaver API documentation lists the relevant mailbox methods, but it does not clearly document the exact response structure for all mailbox state data. Because of that, this integration currently takes a conservative and robust approach and exposes unread-mail sensors instead of per-message entities or built-in message notifications.

Official API documentation:

- https://www.lernsax.de/wws/api.php

## Installation

### HACS

This repository can be added to HACS as a custom repository:

1. Open HACS in Home Assistant
2. Go to the custom repositories section
3. Add this repository URL:
   `https://github.com/alex-LE/ha-lernsax-mailbox`
4. Select the category `Integration`
5. Install **LernSax Mailbox**
6. Restart Home Assistant

### Manual installation

1. Download the latest release from this repository:
   `https://github.com/alex-LE/ha-lernsax-mailbox/releases`
2. Copy `custom_components/lernsax_mailbox` into your Home Assistant configuration directory under:
   `config/custom_components/lernsax_mailbox`
3. Restart Home Assistant

## Configuration

After restarting Home Assistant:

1. Go to **Settings → Devices & Services**
2. Click **Add Integration**
3. Search for **LernSax Mailbox**
4. Enter your LernSax credentials

Each LernSax account should be added as its own config entry.

## Entities

The integration creates the following entities per configured account:

- `sensor.<account>_unread_messages`
- `binary_sensor.<account>_has_unread_mail`

The exact entity IDs depend on the account name generated by Home Assistant.

## Typical usage

This integration intentionally keeps the feature set small and predictable.

Typical follow-up use cases in Home Assistant are:

- push notifications via your own automations
- dashboard cards showing unread messages
- household-specific routing based on the account
- reminders if unread mail exists for a longer period of time
